Architecture Overview

This system is built around one invariant: execution is disposable, decisions are durable. That separation is what makes fairness, retries, and crash recovery safe at scale.

Two-plane model

Control Plane (Redis + API)
  • Owns job state, quotas, and execution ownership
  • Enforces fairness and cost limits before execution
  • Makes retries and crash recovery safe
Data Plane (Workers)
  • Stateless executors
  • Run only while holding a valid lease
  • Can crash without corrupting global state

Core invariants

  • No job runs without paying (quota is charged at claim time)
  • No double execution (only QUEUED jobs can be claimed)
  • No leaked GPUs (execution rights are leased and reclaimable)
  • Failures are boring (crashes turn into safe retries)
  • Fairness is enforced before execution, not retroactively

Why this architecture matters

This is not an agent framework. It is a governance system. Once correctness under concurrency and failure is guaranteed, optimizations like shared heavy context become safe follow-ons.
